Manejo de la pantalla LCD

¿Que es la pantalla LCD?:

Los displays LCD (Liquid Crystal Display), permiten graficar o visualizar los caracteres contemplados en el código ASCII. Este esta gobernado por un microcontrolador el cual dirige todo su funcionamiento siendo este el HD44780.

Además del código ASCII, los displays LCD admiten graficar hasta 8 caracteres diseñados por el desarrollador.

Otra característica fundamental de los LCD, es la conexión del bus de datos, físicamente tienen 8 bits, pero es posible configurar las conexiones con solo 4 bits. La conexión de 8 bits implica una mayor cantidad de cables para su uso, pero la velocidad de trabajo es mayor, por consiguiente la conexión de 4 bits minimiza las conexiones pero disminuye la velocidad de trabajo.

Partes que conforman el display LCD:

  1. Pines de alimentación:

    • Vss: Gnd.
    • Vdd: 5V.
    • Vo/Vee: Corresponde al pin de contraste, el cual se puede regular con un potenciómetro conectado a Vdd.

  2. Pines de control:

    • RS: EL pin RS funciona paralelamente a los pines del bus de datos. Cuando RS es “0” el dato presente en el bus pertenece a un registro de control/instrucción y cuando RS es “1” el dato presente en el bus de datos pertenece a un registro de datos o un carácter.

    • R/W: Este corresponde al pin de Escritura(0) o Lectura(1). Nos permite escribir un dato en la pantalla o leer un dato desde la pantalla.

    • E: Corresponde al pin Enable o de habilitación. Si E = 0 esto quiere decir que el LCD no esta activado para recibir datos, pero si E = 1 se encuentra activo y podemos escribir o leer desde el LCD.

  3. Pines de Bus de Datos:

    • El bus de datos bidireccional esta comprendido desde D0 hasta D7. para realizar la comunicación con el LCD podemos utilizar los 8 bits del bus de datos (desde D0 hasta D7) o empleando los 4 bits mas significativos (desde D4 hasta D7).

  4. DDRAM y CGROM: Son las dos zonas de la memoria del LCD.

    • DDRAM (Display data Ram): Almacena los datos que se desean visualizar por pantalla representados en código de caracteres de 8 bits, es decir, es la memoria donde se almacenan los caracteres a mostrar con su correspondiente posición.

    • CGROM (Character Generator ROM): Es una memoria interna donde se almacena una tabla con los caracteres que podemos visualizar en el LCD. Cuando se le dice al LCD quemuestre la letra “A”, este necesita saber que puntos activar para que se pueda observar una “A”, esta información es la que esta almacenada en la CGROM.

Manejo de la pantalla LCD con AVR Studio

Ahora se tratará el como hacer funcionar la pantalla LCD con el microcontrolador ATmega328P junto con el AVR Studio 5, para esto, se necesitará una librería adicional que se utilizará como un archivo de cabecera principal, con esta librería lograremos trabajar con la pantalla LCD mediante funciones muy sencillas de recordar.

Para descargar el archivo necesario basta con entrar a través del siguiente enlace: LCD_4bits.h

En él se logra descargar el driver necesario para trabajar con solo 4 bits de la pantalla LCD.

Una vez descargada, se verá que se tiene el archivo LCD_4bits.h:

Para utilizar este archivo para el manejo del LCD con AVR Studio, basta con seguir los siguientes pasos:

  1. En el cuadro “Solution Explorer” sobre la carpeta que tiene el nombre del proyecto creado, en este caso se llamo “Manejo_LCD”, hacer click sobre el botón derecho y en las opciones que aparecen, dar “Add” luego “Existing Item…” y luego buscar el archivo LCD_4bits.h descargado previamente
  2. Luego de realizar los pasos anteriores dentro del programa que se esta realizando, será necesario incluir el archivo LCD_4bits.h
    3. copiar código
    #include "LCD_4bits.h"
  3. Una vez incluido el archivo basta con definir los pines correspondientes para el manejo de la pantalla LCD el cual se establece de la siguiente manera:
    4. copiar código
    #define F_CPU 16000000UL // 16 MHz
    /********* Conexiones a la pantalla LCD ****/
    #define D4 eS_PORTD4 // D4 -> PORTD4
    #define D5 eS_PORTD5 // D5 -> PORTD5
    #define D6 eS_PORTD6 // D6 -> PORTD6
    #define D7 eS_PORTD7 // D7 -> PORTD7
    #define RS eS_PORTB5 // RS -> PORTB5
    #define EN eS_PORTB4 // E -> PORTB4
    /*******************************************/
    #include <avr /io.h>
    #include <util /delay.h>
    #include "LCD_4bits.h"

Funciones en LCD_4bits.h

  • Lcd4_Init()

    Esta función inicializa el módulo LCD conectado a los pines del microcontrolador definidos por las siguientes constantes.

    copiar código
    #define D4 eS_PORTD4 // D4 -> PORTD4
    #define D5 eS_PORTD5 // D5 -> PORTD5
    #define D6 eS_PORTD6 // D6 -> PORTD6
    #define D7 eS_PORTD7 // D7 -> PORTD7
    #define RS eS_PORTB5 // RS -> PORTB5
    #define EN eS_PORTB4 // E -> PORTB4

    Estas conexiones deben estar definidas para el correcto funcionamiento de la librería LCD.

    Nota: No olvidar definir estos pines como salida


  • Lcd4_Clear()

    Al llamar esta función se borrará la pantalla LCD

  • Lcd4_Set_Cursor (x, y)

    Esta función establecerá la posición del cursor en la pantalla LCD especificando su fila (x) y columna (y)

  • Lcd4_Write_Char (‘ ’)

    Esta función escribirá un solo carácter en la pantalla LCD y la posición del cursor se incrementará en uno

  • Lcd4_Write_String (“ ”)

    Esta función escribirá una cadena o texto en la pantalla LCD y la posición del cursor se incrementará en la longitud de la cadena más uno

  • Lcd4_Shift_Left() y Lcd4_Shift_Right()

    Estas funciones cambiarán los datos en la pantalla LCD sin cambiar los datos en la RAM de la pantalla.

Ejemplo de código en C

copiar código
#define F_CPU 16000000UL // 16 MHz
/********* Conexiones a la pantalla LCD ****/
#define D4 eS_PORTD4 // D4 -> PORTD4
#define D5 eS_PORTD5 // D5 -> PORTD5
#define D6 eS_PORTD6 // D6 -> PORTD6
#define D7 eS_PORTD7 // D7 -> PORTD7
#define RS eS_PORTB5 // RS -> PORTB5
#define EN eS_PORTB4 // E -> PORTB4
/*******************************************/
#include <avr/io.h>
#include <util/delay.h>
#include "LCD_4bits.h"

int main(void){
DDRD = 0xFF;
DDRB = 0xFF;
int i;
Lcd4_Init ();
while (1) {
Lcd4_Set_Cursor (1,1);
Lcd4_Write_String ("Manejo de la Pantalla LCD");
for(i = 0; i < 15; i ++) {
_delay_ms (200);
Lcd4_Shift_Left ();
}
for(i = 0; i < 15; i ++) {
_delay_ms (200);
Lcd4_Shift_Right ();
}
Lcd4_Clear ();
Lcd4_Set_Cursor (2,1);
Lcd4_Write_Char ('M');
Lcd4_Write_Char ('i');
Lcd4_Write_Char ('c');
Lcd4_Write_Char ('r');
Lcd4_Write_Char ('o');
_delay_ms (2000);
}
}

Reasignar los pines de conexión a la pantalla LCD

Si queremos que los pines establecidos para la conexión a la pantalla LCD sean distintos a los que ya fueron dados y estén a donde nosotros mejor nos parezca basta con cambiar las siguientes líneas de código:

Antes:
copiar código
#define D4 eS_PORTD4 // D4 -> PORTD4
#define D5 eS_PORTD5 // D5 -> PORTD5
#define D6 eS_PORTD6 // D6 -> PORTD6
#define D7 eS_PORTD7 // D7 -> PORTD7
#define RS eS_PORTB5 // RS -> PORTB5
#define EN eS_PORTB4 // E -> PORTB4

Así pues, si queremos, por ejemplo, cambiar la ubicación de los pines del bus de datos de la pantalla LCD al PuertoC del microcontrolador basta con reasignarlos de la siguiente manera:

Después:
copiar código
#define D4 eS_PORTC0 // D4 -> PORTC0
#define D5 eS_PORTC1 // D5 -> PORTC1
#define D6 eS_PORTC2 // D6 -> PORTC2
#define D7 eS_PORTC3 // D7 -> PORTC3
#define RS eS_PORTB5 // RS -> PORTB5
#define EN eS_PORTB4 // E -> PORTB4

Y de esta manera se logra reasignar los pines de conexión a la pantalla LCD según nuestro gusto

Nota: Recordar establecer el puerto utilizado para la comunicación como salida


Mostrar valores numéricos en la pantalla LCD

Resulta importante destacar el hecho que solo se pueden enviar caracteres a la pantalla LCD y no valores numéricos, por lo tanto hay que convertir dicho valor numérico que se mostrara el la pantalla LCD en una cadena de caracteres, para esto hay que incluir el siguiente archivo de cabecera:

copiar código
#include <stdlib.h>

Este archivo posee una serie de funciones que nos permiten convertir valores numéricos en una cadena de caracteres y también en caso contrario, es decir, de cadena de caracteres a valores numéricos

Las funciones que nos interesan son las siguientes:
  • itoa(NumeroAConvertir, Arreglo, BaseNumerica);

    La función itoa convierte un número entero de 8 bits en una cadena de caracteres donde hay que agregarle 3 parámetros en el orden que se muestra:

    1. NumeroAConvertir: Se refiere al número entero que se quiere visualizar en la pantalla LCD el cual será convertido en una cadena de caracteres
    2. Arreglo:Es un arreglo donde se guardará el número entero convertido a cadena, de tipo char con una longitud adecuada para almacenar la cantidad de caracteres del número convertido
    3. BaseNumerica: Es la base en la que se quiere ver el número convertido en la pantalla LCD, si se quiere ver en base binaria este será 2, si se quiere ver en decimal este será 10, 16 para hexadecimal y así.

    • Si el número que se quiere ver es de mas de 8 bits, por ejemplo, 16 bits, se tienen las siguientes funciones:

  • utoa(NumeroAConvertir, Arreglo, BaseNumerica);
  • ultoa(NumeroAConvertir,Arreglo,BaseNumerica);
    • Si el número a convertir es de tipo float o double se tiene la siguiente función:
  • dtostrf(NumeroAConvertir, Tamaño, Precisión ,Arreglo)
    1. NumeroAConvertir: Se refiere al número tipo float que se quiere visualizar en la pantalla LCD
    2. Tamaño:Se refiere al tamaño del número a visualizar en caracteres
    3. Precisión: Se refiere al número de decimales que se desean visualizar
    4. Arreglo: Es un arreglo donde se guardará el número tipo float convertido a cadena, de tipo char con una longitud adecuada para almacenar la cantidad de caracteres del número convertido

Ejemplo de código en C para visualizar números

copiar código
#define F_CPU 16000000UL // 16 MHz
/********* Conexiones a la pantalla LCD ****/
#define D4 eS_PORTD4 // D4 -> PORTD4
#define D5 eS_PORTD5 // D5 -> PORTD5
#define D6 eS_PORTD6 // D6 -> PORTD6
#define D7 eS_PORTD7 // D7 -> PORTD7
#define RS eS_PORTB5 // RS -> PORTB5
#define EN eS_PORTB4 // E -> PORTB4
/*******************************************/
#include <avr/io.h>
#include <util/delay.h>
#include "LCD_4bits.h"
#include <stdlib.h>

int main(void) {
DDRD = 0xFF;
DDRB = 0xFF;
/*** Inicializacion de la pantalla LCD ***/
Lcd4_Init ();
Lcd4_Set_Cursor (1,0);
Lcd4_Write_String ("N entero:");
Lcd4_Set_Cursor (2,0);
Lcd4_Write_String ("N decimal:");
/****************************************/
char Num[20]; // Arreglo donde se almacena el numero convertido a cadena
uint8_t Numero1 = 245; //numero de 8 bits sin signo que se mostrara en la LCD
float Numero2 = 73.49; //numero decimal a mostrar en la LCD
while (1) {
itoa(Numero1, Num, 10); // convierte el numero tipo entero a String con base decimal (10)
Lcd4_Set_Cursor (1,11);
Lcd4_Write_String (Num); // Se muestra el numero convertido
_delay_ms(200);

dtostrf(Numero2, 4, 3, Num); // Convierte el numero tipo float en string con tres decimales
Lcd4_Set_Cursor (2,11);
Lcd4_Write_String (Num); // Se muestra el numero decimal convertido
_delay_ms(200);
}
}
