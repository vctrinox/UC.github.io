Timer 1 en el ATmega328P

Timer1 en el ATmega328P:

Fundamentos

El Timer1 es un temporizador de 16 bits cuyo funcionamiento es sencillo. Cuenta con el registro TCNT1 que se encarga de retener el conteo del temporizador que se incrementa en una unidad en cada ciclo del reloj, este, cuando se utiliza como temporizador, sus valores aumentarán de uno en uno de 0 hasta 65535 (2^16) y luego se desborda (lo que puede producir una interrupción) a lo que, se coloca en 1 un indicador de desbordamiento del temporizador (bit TOV1 ubicado en el registro TIFR1) continuando de nuevo el conteo desde su valor mínimo a su valor máximo

Diagrama de tiempo

En el diagrama anterior podemos apreciar como va incrementando el registro TCNT1. En esta ocasión, se presenta un diagrama de tiempo sin pre escalador (𝑐𝑙𝑘𝐼/𝑂/1) es decir, el registro TCNT1 se incrementará por cada ciclo (𝑇) de reloj del microcontrolador (𝑐𝑙𝑘𝐼/𝑂).

Usos

Cuando es utilizado como contador el temporizador ya no aumenta su valor de uno en uno en cada ciclo de reloj, sino que lo hará mediante el flanco de subida o el flanco de bajada de alguna señal que llegue a un pin en especifico del AVR que afecta directamente al timer1, este pin es identificado como T1 que en el ATmega328P es el bit 6 del puerto D (pin PD5), esto puede variar de acuerdo al microcontrolador AVR utilizado, pero siempre se llamará T1

También se puede cargar un valor inicial en el registro TCNT1 y comenzar la temporización a partir del valor inicial especificado. Otra característica es que se puede establecer un valor en el registro de comparación de salida (OCR1) y cada vez que el TCNT1 alcance ese valor (TCNT1 = OCR1) el bit de comparación de salida se coloca en 1 (OCF1 = 1)

Timer1 como temporizador:

Configuración del Registro TCCR1B

Para programar el Timer1 como temporizador se cuenta con el registro TCCR1B que nos permitirá configurarlo de esta manera

En primera instancia, el timer1 cuenta con lo que se conoce como el pre escalador, esto hace que la frecuencia de trabajo (𝐹𝐶𝑃𝑈) se divida por este pre escalador, con lo que se logra que el timer1 tarde un poco mas en aumentar su valor en una unidad; el pre escalador puede tomar el valor de 1, 8, 64, 256 o 1024;estos valores se eligen programando los bits 0, 1 y 2 del registro TCCR1B, los bits 7 a 3 en este caso se pondrán a 0.

En la siguiente tabla se puede observar los valores que hay que dar a estos bits para obtener los diferentes valores de los pre escaladores para el temporizador timer1, observe que cuando estos bits son todos iguales a 0 el timer1 está deshabilitado

Ejemplo: Uso del Pre escalador

Si el pre escalador utilizado es, por ejemplo, de 64 y la 𝐹𝐶𝑃𝑈 = 16𝑀𝐻𝑧, entonces la frecuencia de trabajo del timer1 𝐹𝑇𝑖𝑚𝑒𝑟1 será de 𝐹𝑇𝑖𝑚𝑒𝑟1 = 𝐹𝐶𝑃𝑈 ∕𝑃𝑟𝑒𝑒𝑠𝑐𝑎𝑙𝑎𝑑𝑜𝑟, el tiempo que tardará ahora el temporizador en aumentar una unidad será la inversa de este valor lo cual se conoce como periodo: 𝑇𝑇𝑖𝑚𝑒𝑟1 = 𝑃𝑟𝑒𝑒𝑠𝑐𝑎𝑙𝑎𝑑𝑜𝑟∕𝐹𝐶𝑃𝑈 por lo que en este caso 𝑇𝑇𝑖𝑚𝑒𝑟1 = 64∕16𝑀𝐻𝑧 donde 𝑇𝑇𝑖𝑚𝑒𝑟1 = 4𝑢𝑠, esto quiere decir que ahora el timer1 aumentará en una unidad cada 4𝑢𝑠 y, en este caso, el registro TCNT1 para ir de 0 a 65535 tardará 65535∗𝑇𝑇𝑖𝑚𝑒𝑟0 = 65535∗4𝑢𝑠 = 262.14 𝑚𝑠; y así con la misma idea para los demás pre escaladores.

Tiempo Máximo del Timer1

De acuerdo a los tiempos que se quieran obtener al utilizar el timer1, habrá que utilizar el pre escalador adecuado así como inicializar el registro TCNT1 también con un valor adecuado, no olvidemos que este registro es de 16 bits y por lo tanto solo puede contener valores enteros entre 0 y 65535

Cabe destacar, en primera instancia, que para establecer un determinado tiempo del temporizador hay que considerar primero cuanto es el máximo tiempo que puede generar el mismo, para así, evitar inconvenientes al momento de establecer el valor inicial del registro TCNT1

Para determinar este tiempo máximo que se puede generar con el temporizador basta con colocar el periodo máximo que se puede generar en el Timer1 a través del máximo valor del pre escalador (1024) y luego multiplicarlo por el máximo valor que este alcanza (65535) de esta manera:

También hay que tomar en cuenta que este tiempo máximo dependerá de la frecuencia de trabajo del MCU

Antes de continuar, aclaremos unas cuantas cosas:

  • Dado que el registro TCNT1 es de 8 bits el rango de valores permitidos deben estar comprendido entre 0 y 65535
  • La selección del pre escalador es mas cuestión de tanteo, se escoge uno, se reemplaza en la ecuación anterior y vemos si el número obtenido esta entre 0 y 65535, de ser así la elección del pre escalador es correcta, de lo contrario habrá que utilizar otro
  • El valor obtenido en el TCNT1 (la mayoría de las veces) no suele ser un número entero, por lo que se tiene que escoger entre el número inmediato superior o el número inmediato inferior

Ejemplo 1: Timer1 como temporizador

En este primer ejemplo utilizaremos los cálculos obtenidos anteriormente para apagar y encender un LED conectado al bit 0 del puerto D (PD0) cada 1 s

copiar código
#define F_CPU 16000000UL // Frecuencia de trabajo del microcontrolador
#include <avr/io.h>

int main(void) {
DDRD = (1 << 0); // bit 0 del PuertoD como salida
PORTD = ~(1 << 0); // Led inicialmente apagado
TCCR1B = 0b00000100; // Preescaler de 256
TCNT1 = 3036; // Valor inicial

while (1) {
if (TCNT1 == 65535) {
PORTD ^= (1 << 0); //Cuando TCNT1 llegue a su valor Max. se invierte el estado del bit
TCNT1 = 3036; // se vuelve a cargar el valor inicial para tener la misma base de tiempo
}
}
}

Ejemplo 2: Timer1 como temporizador con base de tiempo más prolongada

En este segundo ejemplo se muestra como se pueden obtener bases de tiempos mucho mayores al tiempo máximo del timer1, basta con incorporar una variable tipo contador y establecer una base fija de tiempo, con los cálculos anteriores se obtuvo una base de 1 s por lo que, para establecer un retardo de 10 segundos la variable tipo contador tiene que llegar a un total de 10 unidades (1 𝑠 ∗ 10 = 10 𝑠𝑒𝑔𝑢𝑛𝑑𝑜s)

copiar código
#define F_CPU 16000000UL // Frecuencia de trabajo del microcontrolador
#include <avr/io.h>
#include <util/delay.h>

uint8_t Count = 0; // Variable tipo contador

int main(void) {
DDRD = (1 << 0); // bit 0 del PuertoD como salida
PORTD = ~(1 << 0); // Led inicialmente apagado
TCCR1B = 0b00000100; // Preescaler de 256
TCNT1 = 3036; // Valor inicial

while (1) {
if (TCNT1 == 65535) {
Count++; // la variable incrementa cada 1 s
TCNT1 = 3036; // se vuelve a cargar el valor inicial para tener la misma base de tiempo
_delay_ms(5);
}

if (Count == 10) { //Base de tiempo de 10 segundos
PORTD ^= (1 << 0); //Cuando la Variable Count llegue a 10 han transcurrido aprox. 10 segundos y se invierte el estado del bit0
Count = 0; // vuelve a iniciar el conteo de 10 segundos
}
}
}

Timer1 como contador:

Se sabe que el registro TCNT1 del timer1 está formado por 16 bits por lo que se puede contar desde 0 hasta 65535, al utilizar timer1 como contador este no aumentará sus valores o su cuenta con cada ciclo de programa, en este caso el timer1 estará conectado al pin T1 por donde se le hará llegar una señal, la cual al cambiar de estado hará que el timer1 se incremente en una unidad, estos incrementos en su cuenta pueden ser por cada flanco de subida o cada por flanco de bajada de la señal externa, el flanco a utilizar se elige por programa, por ejemplo si se elige el flanco de subida quiere decir que cada vez que la señal que llega al pin T1 pase de un bajo a un alto o de 0 a 1, el registro TCNT1 aumentará en una unidad, de esta manera se le puede utilizar como contador, cuando la señal pase de un alto a un bajo o de 1 a 0 el registro TCNT1 no aumentará su valor o no se incrementará. En la siguiente imagen se muestra el conteo por flanco de subida.

Configuración del Registro TCCR1B

Para utilizar el timer1 como contador, se recurre al mismo registro TCCR1B mencionado anteriormente, hay que configurar los bits 0, 1 y 2 (CS10, CS11 y CS12) los demás bits se pondrán a 0; se tienen dos posibilidades, para que el conteo se realice cuando la señal que llega al pin T1 pase de 0 a 1 o por flanco de subida estos 3 bits se pondrán a 1 esto es CS10=1, CS11=1 y CS12=1; y si el conteo será cuando la señal que llega al pin T1 pase de 1 a 0 o por flanco de bajada solo se pondrán a 1 los bits 1 y 2, mientras el bit0 se deja a 0 esto es CS10=0, CS11=1 y CS12=1.

Ejemplo: Uso del Timer1 como contador

Se utilizará el timer1 como contador, para ello se diseño un circuito tal como se muestra en la imagen, la señal digital será una onda cuadrada de 5Khz generada con la ayuda de un generador de funciones conectado al pin T1 (Pin PD5), se hará que le registro TCNT1 aumente su cuenta por cada flanco de bajada, en 5Khz en cada segundo habrán 5000 flancos de bajada, por lo que en cada segundo el registro TCNT1 aumentará su valor 5000 veces, que es lo mismo que decir que realizará 5000 conteos. Como el TCNT1 puede contar de 0 a 65535 si esto se divide entre 3 (65535/3 = 21845) se hará que un led RGB (Conectado al puerto B) encienda según el conteo que se lleve, si el conteo esta por debajo de 21845 (TCNT1 < 21845) el Led estará encendido de color verde, si está entre 21845 y 43690 (21845 * 2) el Led cambiara a color amarillo, y por ultimo, si el conteo esta por encima de 43690 el Led cambiara a color rojo (La tabla adjunta representa las combinaciones para los distintos colores que se presentan en el LED RGB)

copiar código
#define F_CPU 16000000UL // Frecuencia de trabajo del microcontrolador
#include <avr/io.h>
#include <util/delay.h>

int main(void) {
DDRB = 0xFF; //salidas al Led RGB
PORTB = 0x00; // Led inicialmente apagado
TCCR1B = 0b00000110; // conteo por flanco de bajada
TCNT1 = 0; // valor inicial

while (1) {
if (TCNT1 <= 21845) PORTB = 0x02; // LED verde
if (TCNT1 > 21845 && TCNT1 <= 43690) PORTB = 0x06; // LED amarillo
if (TCNT1 > 43690) PORTB = 0x04; // LED rojo
_delay_ms(100);
}
}

Timer1 modo CTC:

Al utilizar el timer1 en modo comparador ocurre que el registro TCNT1, que es de 16 bits, irá aumentando su valor de uno en uno desde 0 hasta que este se haga igual a un valor almacenado en el registro OCR1A o en el registro ICR1, en el momento que se hacen iguales los valores de los registros TCNT1 y OCR1A o el TCNT1 y el registro ICR1, el registro TCNT1 se pone automáticamente a cero, para nuevamente comenzar a aumentar su valor desde 0 hasta que se vuelva hacer igual al valor almacenado en el registro OCR1A o el registro ICR1

Al mismo tiempo una bandera de comparación es activada (OCF1A o OCF1B según sea el caso) los que nos permitirá manejar interrupciones

A esto se lo conoce como el modo de trabajo CTC, que quiere decir borrado del timer por comparación

Registros involucrados

En el modo de trabajo CTC del timer1 se utilizarán los registros TCCR1A y TCCR1B

Configuración del Registro TCCR1A

Con este registro se elige si el evento a ocurrir como respuesta por comparación a la igualdad entre el registro TCNT1 y el registro OCR1A o el registro ICR1, será por el pin OC1A u OC1B

Los bits 7 y 6 controlan el comportamiento del pin de comparación de salida (OC1A o bien el pin PB1 del ATmega328P), para elegir el tipo de evento que ocurrirá cuando se iguale el registro TCNT1 con el registro OCR1A, en el caso de utilizar el pin OC1A para las combinaciones de los bits 7 y 6 se tiene la siguiente tabla, si ambos bits son 0 la opción OC1A del pin no está disponible por lo que el pin se comporta como una entrada/salida digital normal

Si por ejemplo el bit7 se pone a 0 y el bit6 a 1 entonces el estado del pin OC1A cambiará cada vez que la comparación ocurra,si el bit7 y el bit6 se ponen a 1 ocurrirá que si el pin OC1A estaba a 0 este se pondrá a 1, pero si estaba a 1 este permanecerá igual

Estos bits controlan el comportamiento del pin de comparación de salida (OC1B o bien el pin PB2 del ATmega328P) para elegir el tipo de evento que ocurrirá cuando se iguale el registro TCNT1 con el registro OCR1A, en el caso de utilizar el pin OC1B para las combinaciones de los bits 5 y 4 se tiene la siguiente tabla, si ambos bits son 0 la opción OC1B del pin no está disponible por lo que el pin se comporta como una entrada/salida digital normal

Si por ejemplo el bit5 se pone a 0 y el bit4 a 1 entonces el estado del pin OC1B cambiará cada vez que la comparación ocurra, si el bit5 y el bit4 se ponen a 1 ocurrirá que si el pin OC1B estaba a 0 este se pondrá a 1, pero si estaba a 1 este permanecerá igual

Los bits 3 y 2 no se utilizan por lo que se les suele poner a 0

El bit1 y el bit0 tienen que ser puesto a 0 esto, junto con el bit3 puesto a 1 y el bit4 puesto a 0 del registro TCCR1B se logra utilizar el modo CTC

Configuración del Registro TCCR1B

Con este registro se elige el pre escalador a utilizar, para obtener en cuanto tiempo se quiere que el registro TCNT1 se iguale al registro OCR1A o el registro ICR1

En la siguiente tabla se puede observar los valores que hay que dar a estos bits para obtener los diferentes valores de los pre escaladores, observe que cuando estos bits son todos iguales a 0 el timer1 está deshabilitado, los bit 7, 6 y 5 se colocan a 0 dado que no se utilizarán, los bit 4 y 3 se configuran como se menciono en la lamina anterior

Configuración del registro OCR1A

En el registro OCR1A se cargará el valor hasta el cual se quiere que llegue el registro TCNT1 en el modo CTC, por lo que hay que obtener el valor adecuado a cargar en el registro OCR1A, esto se realiza mediante la siguiente expresión

El registros OCR1A es de 16 bits por lo que el valor obtenido mediante la ecuación anterior tendrá que ser un número entero comprendido entre 0 y 65535, de no ser así el valor no será válido

Por ejemplo, para lograr que la comparación ocurra cada 1 s, utilizando un pre escalador de 1024 será necesario cargar el registro OCR1A con el valor de:

15625 sería el valor a cargar en el registro OCR1A para que cada 1 s el valor del registro TCNT1 llegue a 15625 y por comparación se produzca un evento en el pin OC1A u OC1B, tras lo cual el registro TCNT1 se pondrá automáticamente a 0 para volver a aumentar su valor nuevamente hasta 15625 en un lapso de 1 s y se vuelva ha repetir el proceso

Ejemplo: Uso del Timer1 como comparador

Para el ejemplo del uso del timer1 en modo CTC, se utilizará el pin PB2 o bien el OC1B del ATmega328P como el pin en el que ocurrirá un evento, que será cada vez que ocurra la igualdad entre el registro TCNT1 y el registro OCR1A donde el estado de este pin OC1B cambie, este pin hay que configurarlo como una salida digital mediante el DDRB correspondiente, se hará que el pin OC1B cambie de estado cada 1 s, esto será observado mediante un LED conectado a este pin

Circuito en proteus:

Utilizando los cálculos anteriores, cargamos el registro OCR1A = 15625, del registro TCCR1A hay que poner a 1 su bit4 para que el evento por cambio de estado sea por el pin OC1B, y también hay que poner a 0 su bit1 y bit0 para hacer trabajar el timer1 en modo CTC, sus demás bits se ponen a 0

TCCR1A = 0b00010000; //cambio estado en el pin OC1B, modo CTC

Del registro TCCR1B se colocará a 1 su bit2, a 0 el bit1 y a 1 el bit0 para tener el pre escalador de 1024, y luego a 1 el bit3 y a 0 el bit4 para el modo CTC

TCCR1B = 0b00001101; //prescaler de 1024, modo CTC

El código completo queda de la siguiente manera:

copiar código
#define F_CPU 16000000UL // Frecuencia de trabajo del microcontrolador
#include <avr/io.h>
#include <util/delay.h>

int main(void) {
DDRB = (1 << 2); // PB2 como salida
TCCR1A = 0b00010000; //cambio estado en el pin OC1B, modo CTC
TCCR1B = 0b00001101; //prescaler de 1024, modo CTC
OCR1A = 15625; // valor a comparar
while (1) {
_delay_ms(50);
}
}
