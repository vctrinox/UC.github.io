Manejo del Display 7 Segmentos

¿Qué son los Display 7 segmentos?

Un LED o Diodo Emisor de Luz, es un diodo óptico de unión tipo PN que emite energía luminosa en forma de fotones.

La emisión de estos fotones ocurre cuando la unión del diodo esta polarizada por un voltaje externo de tal forma que la corriente fluya a través de su unión, en electrónica, este proceso se llama Electroluminiscencia

La principal ventaja de los diodos emisores de luz es que debido a su pequeño tamaño, varios de ellos pueden conectarse entre sí dentro de un paquete pequeño y compacto que produce lo que generalmente se llama un Display 7 Segmentos

El Display 7 Segmentos consta de siete LEDs (de ahí su nombre) dispuestos de forma rectangular como se muestra en la figura. Cada uno de los siete LEDs se denomina “segmento” porque cuando se iluminan, el segmento forma parte de un digito numérico (tanto decimal como hexadecimal) que se mostrará. En ocasiones se utiliza un octavo LED adicional dentro del mismo paquete, lo que permite la indicación de un punto decimal (DP)

A cada uno de los siete LEDs en el Display se le asigna un segmento posicional con uno de los extremos de sus pines de conexión que se saca directamente del paquete de plástico. Estos pines están etiquetados desde la “a” hasta la “g” que representa a cada LED de manera individual. Los otros extremos de los LEDs están conectados entre sí para formar un pin común

Display Cátodo Común y Ánodo Común

El pin común del Display se usa generalmente para identificar que tipo de Display 7 segmentos es. Como cada LED tiene dos pines de conexión, Ánodo y Cátodo, existen dos tipos de Display 7 segmentos:

  • Cátodo Común (CC)
  • Ánodo Común (AC)
Cátodo Común (CC):

En este Display todas las conexiones de cátodo de los segmentos se unen a la lógica “0” o tierra. Los segmentos individuales se iluminan mediante la aplicación de una señal “Alta” o un “1” lógico a través de una resistencia limitadora de corriente

Ánodo Común (AC):

En este Display, todas las conexiones de ánodo de los segmentos se unen a la lógica “1”. Los segmentos individuales se iluminan aplicando una señal de tierra, lógica “0” o una señal “Baja” a través de una resistencia limitadora de corriente

Display de 4 dígitos

Se puede utilizar un modulo compacto de 4 dígitos multiplexandolos, esto significa, que se muestra un digito a la vez, esto es: primero se enciende un único Display, luego, al transcurrir un periodo de tiempo muy corto, se apaga este Display e inmediatamente se enciende el que le sigue mostrando un número decimal diferente, este proceso se repite indefinidamente. Esto sucede a tal velocidad que el ojo humano no puede diferenciar entre que Display se esta apagando o encendiendo, sino que, observará que todos los Display se encuentran encendidos

Así pues, en este sentido, se necesitarían menos pines de conexión para manejar más de un Display. Si se quiere trabajar con dos, se necesitarán 2 pines de control y los 7 correspondientes a los 7 segmentos del Display, si se trabaja con 4, se necesitarán 4 pines de control más los 7 pines correspondientes, y así sucesivamente.

Entonces, si, por ejemplo, se desea visualizar en el Display de 4 dígitos los números 1234, en primera instancia se habilita (enciende) el primer Display mientras el resto permanecerá apagado, mostrando, de esta manera, el digito correspondiente, en este caso el número 1, luego, este Display se apagará e inmediatamente se habilitará el siguiente Display mostrando el número 2 y así con el resto de la secuencia. Manejo del Display

Lo primero que tenemos que considerar al momento de utilizar el Display 7 segmentos es la ubicación del mismo en el microcontrolador, de esta manera, se establece el puerto a utilizar y con esto establecer la asignación de los 7 segmentos

Para este primer ejemplo se utilizaran 7 bits del PuertoD del microcontrolador para establecer la conexión, luego, a cada uno de estos bits se le asignaran los segmentos desde la “a” hasta la “g”, y, escogiendo un Display CC, se realiza la tabla de la verdad correspondiente a continuación

Ya establecido lo anterior, es necesario elaborar un Arreglo donde estén almacenados los diez dígitos correspondientes en forma ordenada. De esta manera:

copiar código
unsigned char Dig[10] = {0x3F,0x06,0x5B,0x4F,0x66,0x6D,0x7D,0x07,0x7F,0x6F};

Ejemplo de código en C parte 1

copiar código
#define F_CPU 16000000UL // Frecuencia de trabajo del microcontrolador
#include <avr/io.h>
#include <util/delay.h>

/***************** Declaracion de variables ******************************/
unsigned char Dig[10] = {0x3F,0x06,0x5B,0x4F,0x66,0x6D,0x7D,0x07,0x7F,0x6F};
uint8_t D0 = 0; // Variable para recorrer el arreglo
/*************************************************************************/

/***************** Configuracion *************************/
void configuracion() {
DDRD = 0xFF; // Puerto D como salida
PORTD = 0; // Segmentos del display apagados
}
/*********************************************************/

/********** funcion para mostrar en el display *************/
int mostrar() {
PORTD = Dig[D0]; //Muestra en el PuertoD los digitos de la tabla según el valor de D0
_delay_ms(10);
}
/*********************************************************/

/************** Programa principal **********************/
int main(void) {
configuracion();
while (1) {
mostrar(); // Llama a la funcion para mostrar los digitos en el Display
D0 += 1; // D0 = D0 + 1; D0 se utiliza para recorrer la tabla
_delay_ms(300);
if (D0 >= 10) D0 = 0; // Si se recorrio toda la tabla D0 se reinicia a 0 para un nuevo comienzo
}
}

Reasignación del puerto de salida para manejar el Display 7 segmentos

Ahora bien, que sucede si queremos cambiar la posición del Display en el microcontrolador, si observamos cuidadosamente la placa Arduino, nos daremos cuenta que solo el PuertoD cuenta con los 7 bits necesarios para ocupar todos los segmentos, mientras que, el PuertoC y el PuertoB cuentan solo con 6 bits disponibles para la conexión, lo cual, hace necesario la utilización de un bit de un puerto diferente al que será utilizado

Para este siguiente ejemplo será utilizado el PuertoB para los segmentos del Display desde el a hasta la f y para el segmento g será utilizado el bit 6 del PuertoD (PD6)

Si nos damos cuenta, se esta utilizando exactamente la misma tabla que en el ejemplo anterior

Para lograr nuestro objetivo es necesario saber cuando se requiere encender el segmento g para que, de esta manera, se logre habilitar el bit que se encuentra en un puerto diferente y lograr el efecto deseado

Ejemplo de código en C parte 2

copiar código
#define F_CPU 16000000UL // Frecuencia de trabajo del microcontrolador
#include <avr/io.h>
#include <util/delay.h>

/***************** Declaracion de variables ******************************/
unsigned char Dig[10] = {0x3F,0x06,0x5B,0x4F,0x66,0x6D,0x7D,0x07,0x7F,0x6F};
uint8_t tempo, D0 = 0; // tempo se usara como variable temporal
/*************************************************************************/

/***************** Configuracion *************************/
void configuracion() {
DDRD = 0xF0; // Puerto D para usar el segmento g
DDRB = 0xFF; // PORTB como salida
PORTB = 0; // Segmentos del display apagados
PORTD = 0; // Segmentos del display apagados
}
/*********************************************************/

/********** funcion para mostrar en el display *************/
int mostrar() {
tempo = Dig[D0] & 0b01000000; // enmascara el segmento g, con esto se logra determinar cuando es necesario utilizar el segmento g
if (tempo == 0x40) PORTD = (1 << PORTD6); // enciende el segmento g cuando este sale en el arreglo
else PORTD = (0 << PORTD6); // de lo contrario este permanecera apagado
PORTB = Dig[D0]; // se muestran los dígitos correspondientes en el Display a través del puerto B y con el segmento g ubicado en el bit 6 del puerto D
_delay_ms(10);
}
/*********************************************************/

/************** Programa principal **********************/
int main(void) {
configuracion();
while (1) {
mostrar(); // Llama a la funcion para mostrar los digitos en el Display
D0 += 1; // D0 = D0 + 1; D0 se utiliza para recorrer la tabla
_delay_ms(300);
if (D0 >= 10) D0 = 0; // Si se recorrio toda la tabla D0 se reinicia a 0 para un nuevo comienzo
}
}

Ejemplo de código en C: multiplexación de displays

En este ejemplo se ilustra como realizar la multiplexación con dos Displays

copiar código
#define F_CPU 16000000UL // Frecuencia de trabajo del microcontrolador
#include <avr/io.h>
#include <util/delay.h>

/***************** Declaracion de variables ******************************/
unsigned char Dig[10] = {0x3F,0x06,0x5B,0x4F,0x66,0x6D,0x7D,0x07,0x7F,0x6F};
uint8_t tempo, D0 = 0, D1 = 0; // tempo se usara como variable temporal
/*************************************************************************/

/***************** Configuracion *************************/
void configuracion() {
DDRD = 0xFF; // Puerto D como salida
DDRB = 0xFF; // Puerto B como salida
PORTB = 0; // Segmentos del display apagados
PORTD = 0; // Segmentos del display apagados
}
/*********************************************************/

/********** funcion para mostrar en el display *************/
int mostrar() {
PORTB = (1 << PORTB0); // Selecciona el display D1
PORTD = Dig[D0];
_delay_ms(40);
PORTB = (1 << PORTB1); // Selecciona el display D0
PORTD = Dig[D1];
_delay_ms(40);
}
/*********************************************************/

/************** Programa principal **********************/
int main(void) {
configuracion();
D0 = 1;
D1 = 2;
while (1) {
mostrar(); // Llama a la funcion para mostrar los digitos en el Display
_delay_ms(40);
}
}
