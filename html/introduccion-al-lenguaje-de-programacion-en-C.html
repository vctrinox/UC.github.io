Introducción al lenguaje de programación en C con AVR Studio

Estructura básica de un programa elaborado en C

Básicamente, toda programación elaborada en lenguaje C tendrá la siguiente estructura:

  • La función Main() hace referencia al inicio del programa principal
  • Un bloque establece y delimita el cuerpo de las funciones y algunas sentencias mediante llaves { }
  • Toda sentencia o instrucción escrita en el programa debe finalizar con un punto y coma (;).

El primer renglón de programación lo ocupan las Directivas del pre-procesador, siendo estas un conjunto de archivos que consulta el compilador antes de construir el archivo hexadecimal (.hex) que, posteriormente será grabado en la memoria del microcontrolador. Las directivas siempre comienzan con el carácter denominado numeral (#)

Por ejemplo, si estamos por empezar un nuevo proyecto en el IDE AVR Studio 5, lo primero que se nos muestra al crear el proyecto es el archivo de cabecera: #include <avr/io.h>

  • Este archivo de encabezado incluye las definiciones de Entrada/Salida (I/O) apropiadas para el dispositivo especificado por el usuario.
  • A estos archivos se les conoce comúnmente como Librerías

El segundo renglón de la programación lo ocupa las Declaraciones, estas hacen referencia a porciones de memoria que almacenan un valor. Estos valores suelen ser Variables o Constantes, ambas dependen del tipo de dato.

Como añadir comentarios

Los comentarios se inician con doble barra diagonal ( // ) los mismos se usan para aclarar cosas durante la edición del programa. Los comentarios no se transmiten a la memoria del microcontrolador, por lo que pueden emplearse tantas veces sea necesario.

Si bien se busca establecer una sección muy amplia como comentario, este deberá comenzar y finalizar con una barra y un asterisco ( /* comentario extenso */ ) como se muestra en la siguiente imagen

Declaración de variables

No se puede utilizar una variable en el programa si antes no se ha declarado. La forma general más simple de hacerlo es la siguiente:

Tipo_de_Dato MyVar;

Aqui se especifica lo siguiente:

  • Tipo_de_Dato: Corresponde al tipo de dato con el que se quiere trabajar
  • MyVar: Nombre de la variable a utilizar (siempre y cuando no sea una palabra reservada por el compilador)

Ejemplo:

unsigned int temp; //Variable tipo entero sin signo (16 bits)
signed short a; // Varaiable tipo entero con signo (8 bits)
unsigned short k; // Varaiable tipo entero sin signo (8 bits)
unsigned char c; // Varaiable tipo entero sin signo (8 bits)

También es posible declarar varias variables del mismo tipo, separándolas con comas, además de inicializarlas al momento de ser declararlas. De la siguiente manera:

unsigned int a, b, c; // Variables a, b y c declaradas tipo entero de 16 bits sin signo
short j = 0, k = 1; // Variables j y k declaradas tipo short, donde j se inicializa en 0 y k en 1 (valor inicial)

Operadores por bit
uint8_t m, n; // variables de 8 bits

/*dado que se realiza la operacion AND entre 0xF0 y 0x0F el resultado sera 0x00*/
m = m | 0x24; // Después de la operacion OR m será 0x24

Cuando una variable se desplaza hacia un lado, los bits que salen se pierden y los bits que entran por el otro lado son siempre ceros. Es por esto que en la última sentencia, m = m << 8, el resultado es 0x00.

Sentencia

While

Esta sentencia se utiliza cuando queremos que una sección de nuestro programa se repita mientras se cumple una condición. De no cumplirse la condición, se ignora esta sentencia y continua con el resto del programa

Estructura:
while (condicion)
{
// Bloque de sentencias a ejecutar mientras la condicion se cumpla
}
Ejemplo:
while (1)
{
PORTB = 0b00000001; // Bit cero del PuertoB Activado
_delay_ms(500); // retardo de 500ms
PORTB = 0b00000000; // Bit cero del PuertoB apagado
_delay_ms(500); // retardo de 500ms
}

El número 1 implementado como condición nos indica que esa sección del programa se repetirá indefinidamente dado que la condición siempre será verdadera

Sentencia

Do … While

A diferencia del while la sentencia Do … While nos permite tener por lo menos una iteración, ya que la condición se comprueba al final del bucle

Estructura:
do
{
// Bloque de sentencias que se ejecutan al menos una vez
} while (condicion)
Ejemplo:
do
{
mostrar_numero(); // funcion definida en el programa
_delay_ms(500); // retardo de 500ms
i++; // Variable declarada que se incrementa en una unidad
} while (i < 10);// Condición para que se repita el ciclo, es decir, mientras la variable i sea menor a 10 el bucle se repetira

Sentencia

for

El bucle for permite la inicialización automática y el incremento de una variable tipo contador.

Estructura:
for (Inicialización; Condición; Incremento/Decremento)
{
// Bloque de sentencias
}
Ejemplo:
for (i = 0; i < 10; i++)
{
printf ("hola\n"); // Se imprime 10 veces hola
}

Aunque la sentencia for permite ciertas variaciones, generalmente la Inicialización se utiliza para establecer el valor del comienzo de una variable. La Condición normalmente es una sentencia relacional que compara la variable contador con un valor de terminación

Sentencia

if … else

Ambas son sentencias independientes, pero, por lo general, suelen trabajar en conjunto

Estructura:
if (Condición/Condiciones)
{
//Bloque de sentencias 1
}
else
{
//Bloque de sentencias 2
}

De cumplirse la condición del if se ejecutará lo que se encuentre en el bloque de sentencias 1, de lo contrario, se ejecutará lo que se encuentre en el bloque de sentencias 2 si esta existe

Ejemplo:
if (PINB & (1 << 0)) // Si el pin PB0 es 1 ...
{
PORTB = (1 << PORTB5) & ~(1 << PORTB4); // Se enciende PB5 y se apaga PB4
}
else // si el pin PB0 no es 1 ...
{
PORTB = (1 << PORTB4) & ~(1 << PORTB5); // se enciende PB4 y se apaga PB5
}

Sentencia

breack

break se utiliza para salir de un bucle Do, For o while saltándose la condición normal del bucle. También se utiliza para salir de la sentencia switch

Ejemplo:
int main(void) { //Programa principal
int m;
for (m = 0; m < 100; m++) // El ciclo for normalmente deberia terminar una vez se cumpla la condicion m < 100 es decir, cuando m = 99
{
_delay_ms(500);
if (m == 10) break; // Si m = 10 el ciclo for se termina (no se realizan mas iteraciones) y se continua con el resto del programa
}
}

Sentencia

switch … case

Es la sentencia de bifurcación a varias direcciones del programa. Se utiliza a fin de ejecutar un camino de los distintos posibles.

Se escribe entre paréntesis después de switch la expresión a ser analizada

Estructura:
switch (Variable a analizar para la bifurcación)
{
case (Contante 1):
Conjunto de sentencias 1;
break;

case (Contante 2):
Conjunto de sentencias 2;
break;
.
.
.
case (Contante n):
Conjunto de sentencias n;
break;

default : Sentencias por defecto;
}

Cada conjunto de sentencias puede estar formado por una o varias sentencias. Si no se cumple ninguno de los “case” y se requiere hacer algo por defecto, dicha acción se programa mediante la instrucción default la cual es opcional

Ejemplo:
copiar código
int main(void) { //Programa principal
unsigned short LED = 0; // Variable a analizar
DDRB = 0xFF; // Puerto B como salida

while (1) { // Ciclo infinito
switch (LED) //Para cada caso de la variable LED (que es una variable de tipo contador) se encenderá un bit del PuertoB
{
case (0): PORTB = (1 << PORTB0); break; // Si la variable LED es igual a cero se ejecuta esta sentencia
case (1): PORTB = (1 << PORTB1); break; // Si la variable LED es igual a uno se ejecuta esta sentencia
case (2): PORTB = (1 << PORTB2); break; // Si la variable LED es igual a dos se ejecuta esta sentencia
case (3): PORTB = (1 << PORTB3); break; // Si la variable LED es igual a tres se ejecuta esta sentencia
case (4): PORTB = (1 << PORTB4); break; // Si la variable LED es igual a cuatro se ejecuta esta sentencia
case (5): PORTB = (1 << PORTB5); break; // Si la variable LED es igual a cinco se ejecuta esta sentencia
default: PORTB = 0x00; // En caso de que la variable LED no tenga ninguno de los valores anteriores se ejecuta esta sentencia
}
_delay_ms(1000);
if (LED == 6) LED = 0; // Si la variable llego a 6 se reinicia a cero
else LED++; // Si no ha llegado se incrementa una unidad
}
}

Sentencia

return

Precisamente, la manera que tiene una función para devolver un valor es mediante la sentencia return. Cuando una función se encuentra con una sentencia return se devuelve a la rutina de llamada inmediatamente y las sentencias posteriores a return no se ejecutan

Ejemplo:
unsigned int sum(unsigned short A, unsigned short B) // Funcion para sumar
{
return (A + B); // devuelve la sumatoria de los parametros A y B
}

int main(void) { //Programa principal
a = sum(1, 10); // invoca la funcion suma con los parametros A = 1 y B = 10
b = sum(15, 6); // invoca la funcion suma con los parametros A = 15 y B = 6
c = sum(100, 25); // invoca la funcion suma con los parametros A = 100 y B = 25
/** los resultados seran: a = 11, b = 21 y c = 125 **/
}

Ejemplo de sentencias simples

Para escribir el programa de una manera más sencilla y utilizando menos lineas de código existen equivalencias dónde solo la función de alguna condición se ejcuta en una sola linea de código, de esta manera:

Todas estas expresiones …
/************************* Expresion 1 *************************/
if (a > b) // si a es mayor que b
{
a = 0; // a la variable "a" se le asigna cero
}
/************************* Expresion 2 *************************/
if (a == b) // si a es igual a b
{
a++; // a = a + 1; a se incrementa en una unidad
}
else // de lo contrario
{
b--; // b = b - 1; b se decrementara una unidad
}
/************************* Expresion 3 *************************/
while (a >= b) //mientras "a" sea mayor o igual que "b"
{
a = a + b; // suma "a" con "b" y actualiza el valor de "a"
}
/************************* Expresion 4 *************************/
for (i=0; i <= 10; i++) // mientras i sea menor o igual a 10 se incrementa en una unidad i
{
a = a*2; // y se multiplica la variable "a“ por 2 e inmediatamente se actualiza la variable "a"
}
Son equivalentes a estas expresiones:
/************************* Expresion 1 *************************/
if (a > b) a = 0; si a es mayor que b a la variable "a" se le asigna cero

/************************* Expresion 2 *************************/
if (a == b) a++; // si a es igual a b ... a = a + 1; a se incrementa en una unidad
else b--; // de lo contrario ... b = b - 1; b se decrementara una unidad

/************************* Expresion 3 *************************/
while (a >= b) a = a + b; //mientras "a" sea mayor o igual que "b" ... suma "a" con "b" y actualiza el valor de "a"

/************************* Expresion 4 *************************/
for (i=0; i <= 10; i++) a = a*2; // mientras i sea menor o igual a 10 se incrementa en una unidad i y se multiplica la variable "a" por 2 e inmediatamente se actualiza la variable "a"

Esto se puede hacer siempre y cuando la sentencia que se vaya a realizar sea solo una, si se necesita realizar más de una sentencia se deben colocar los corchetes { }

Funciones en C

Las funciones son los bloques constructivos fundamentales en C. Todas las sentencias deben encontrarse dentro de funciones. Las funciones deben ser declaradas y definidas antes de ser utilizadas

Estructura de una función con parámetros:
Tipo_de_Dato Nombre_Func(Tipo_de_Dato Parametro1, Tipo_de_Dato Parametro2, ...)
{
// Cuerpo de la función/ Sentencias
return algunDato; // esto es necesario solo si la función retornará algún valor
}
  • Nombre_Func: es el nombre que le damos a la función preferiblemente uno fácil de recordar
  • Tipo_Dato: es un tipo de dato que identifica el parámetro de salida, puede ser tipo entero sin signo, con signo o tipo flotante
  • Parametro1, Parametro2 (pueden existir mas): son las variables que recibirán los datos que se pasan a la función

Existen también, funciones sin parámetros, esto quiere decir que son funciones que no reciben ni devuelven ningún valor y cuya estructura se reduce a lo siguiente:

Estructura de una función sin parámetros:
void Nombre_Func(void)
{
// Bloque de sentencias
}

la palabra void nos indica que la función no devuelve ningún valor

Ejemplo de funciones con parámetros (por valor) y sin parámetros

Funciones sin parámetros:

Para el siguiente ejemplo de una función sin parámetros podemos observar que se declaró la función “Configuracion_Inicial” el cual se encarga de configurar los puertos B y D del microcontrolador, luego, esta función es llamada en el programa principal “main” donde se ejecutará dicha función declarada

Ejemplo de una función sin parámetros:
void Configuracion_Inicial() {
DDRB = 0xFF; // Declara el PuertoB como salida
DDRD = 0x00; // Declara el PuertoD como entrada
}

int main(void) {
Configuracion_Inicial(); // Convoca a la funcion "Configuracion_Inicial", aqui es donde se configuran los puertos B y D
}
Funciones con parámetros (por valor):

Según los parámetros de entrada de una función, estos se dividen en dos grupos:

  • Parámetros por valor
  • Parámetros por referencia

En este caso de estudio nos basaremos en los parámetros por valor ya que estos son los mas utilizados.

Ejemplo de una función con parámetros (por valor):
unsigned int sum(unsigned short A, unsigned short B) // Funcion con los parametros A y B
{
return (A + B); // devuelve la sumatoria de los parametros A y B
}

int main(void) { //Programa principal
a = sum(1, 10); // invoca la funcion suma con los parametros A = 1 y B = 10
b = sum(15, 6); // invoca la funcion suma con los parametros A = 15 y B = 6
c = sum(100, 25); // invoca la funcion suma con los parametros A = 100 y B = 25
/** los resultados seran: a = 11, b = 21 y c = 125 **/
}

Importante: Para llamar a una función con parámetros es importante respetar el orden y el tipo de los parámetros que esta recibe. El primer valor pasado (escrito) corresponde al primer parámetro de entrada; el segundo valor, al segundo parámetro; y así tantas veces como parámetros tenga la función

Variables: Variables locales y variables globales

Variables globales:

Las variables declaradas fuera de todas las funciones y antes de sus implementaciones tienen carácter global y se pueden acceder a ellas desde cualquier función o parte del programa

Variables locales:

Las variables declaradas dentro de una función, incluyendo las variables que son pasadas como parámetros, tienen ámbito local. Ellas solo podrán ser accedidas desde el cuerpo de dicha función.

De esta manera, pueden existir dos o más variables con el mismo nombre, siempre y cuando estén en diferentes funciones. Cada variable pertenece a su función y no tiene nada que ver con las variables de otra función, por más que tengan el mismo nombre.

Nota: A diferencia de las variables globales, las variables locales tienen almacenamiento temporal, es decir, se crean al ejecutarse la función y se destruyen al salir de ella.

Cuando se llama a una función sus variables locales se crearán en ese momento y cuando se salga de la función se destruirán. Se entiende por destruir al hecho de que la locación de memoria que tenía una variable será luego utilizada por el compilador para otra variable local (así se mejora el uso de la memoria). Como consecuencia, el valor de las variables locales no será el mismo entre llamadas de función

void Var() {
int a; // Se declara la variable "a"
a++; // se incrementa la variable en una unidad
}

Cada vez que se llame a la función Var se crea la variable “a”, luego se incrementa y, al terminar de ejecutarse la función, se destruye.

Entonces, si, por ejemplo, se llama la función Var 10 veces a lo largo del programa no necesariamente quiere decir que la variable “a” se haya incrementado a un total de 10 veces

Variables estáticas

Para que una variable tenga una locación de memoria independiente y su valor no cambie entre llamadas de función tenemos dos caminos: o la declaramos como global, o la declaramos como local estática

Una variable se hace estática anteponiendo a su declaración el especificador static. Por defecto las variables estáticas se auto inicializan a 0, pero se le puede dar otro valor en la misma declaración (dicha inicialización solo se ejecuta la primera vez que se llama a la función), como se muestra a continuación:

static int var1; // Variable static (inicializada a 0 por defecto)
static int var2 = 50; // Variable static inicializada a 50
Ejemplo:
void Var() {
static int a = 5; //Variable local estática inicializada a 5
a++; // se incrementa la variable en una unidad
}

int main(void) {
int i; // Se declara la variable i
for (i = 0; i <= 10; i++) Var(); // Este segmento de código llama 10 veces a la funcion Var
/**Ahora la variable "a" si deberá valer 15 (a = 15 dado que se inicializa en 5)**/
}

Variables volátiles

La palabra clave volátil esta destinada a evitar que el compilador aplique cualquier optimización en las variables que puedan cambiar en formas que el compilador no puede determinar

Las variables declaradas como volátiles se omiten de la optimización porque sus valores se pueden cambiar por código fuera del alcance del código actual en cualquier momento. El sistema siempre lee el valor actual de una variable volátil desde la ubicación de la memoria en lugar de mantener su valor en el registro temporal en el punto en el que se solicita

Si no usamos el calificador volátil pueden surgir los siguientes problemas:

  • El código puede no funcionar como se esperaba cuando la optimización esta activa
  • El código puede no funcionar como se esperaba cuando las interrupciones están habilitadas y utilizadas
Ejemplo:
volatile int count; // count es ahora una variable global tipo volátil

}

Arrays o Matrices

Una matriz se utiliza para almacenar una colección secuencial de elementos del mismo tipo de tamaño fijo

En lugar de declarar variables individuales, se puede declarar una matriz para representar cada una de ellas, con lo cual, para acceder a un elemento especifico de la matriz se utiliza el número del índice encerrado entre corchetes [ ]

Los Arrays pueden tener N elementos

Para declarar una matriz el usuario especifica el tipo de elementos y la cantidad de elementos a utilizar, de esta manera:

Tipo_de_Dato Nombre_de_la_matriz [ numElementos ];

Esto es lo que se conoce como una matriz unidimensional.

  • NunElementos debe ser una constante tipo entero mayor que cero donde se define la cantidad de elementos que tendrá la matriz
  • Tipo_de_Dato puede ser cualquier tipo de dato
Ejemplo:
unsigned char letras[5]; // Matriz que contiene 5 elementos tipo unsigned char
int Dig[8]; // Matriz de 8 elementos tipo int

Si, por ejemplo, quisiéramos asignar a cada uno de los elementos de la matriz Letras los caracteres desde la ‘a’ hasta la ‘e’, lo podríamos hacer individualmente así:

Asignación de elementos a la matriz:
letras[0] = 'a'; // Aquí al índice 0 se le asigna la variable "a"
letras[1] = 'b'; // Aquí al índice 1 se le asigna la variable "b"
letras[2] = 'c'; // Aquí al índice 2 se le asigna la variable "c"
letras[3] = 'd'; // Aquí al índice 3 se le asigna la variable "d"
.
.

Se puede inicializar los elementos de una matriz uno por uno usando una sola declaración de la siguiente manera:

int Dig[8] = {1, 2, 3, 4, 5, 6, 7, 8}; //Matriz de 8 elementos ya inicializados tipo int

El número de valores entre llaves { } no puede ser mayor al número de elementos que declaramos para la matriz entre corchetes [ ]

Si se omite el tamaño de la matriz (dejando los corchetes vacíos) se crea una matriz lo suficientemente grande como para contener todos los elementos que están siendo inicializados

int Dig[] = {1, 2, 3, 4, 5, 6, 7, 8}; //Matriz de 8 elementos tipo int

Para acceder a un elemento de la matriz se coloca el índice del elemento entre corchetes después del nombre de la matriz:

A = Dig[4]; // se toma el quinto elemento de la matriz Dig y se almacena en la variable A
// A = 5

Punteros

Algunas tareas de programación en C se realizan mas fácilmente con punteros, y otras tareas, como la asignación dinámica de memoria, no se pueden realizar sin usar punteros

Un puntero es una variable cuyo valor es la dirección de otra variable, es decir, la dirección directa de la ubicación en la memoria donde dicha variable es almacenada. Al igual que cualquier variable o constante, se debe declarar antes de ser utilizado para almacenar cualquier dirección de alguna variable


Forma general de declarar una variable tipo puntero:
Tipo_de_Dato * nombre_del_puntero;
  • nombre_del_puntero es el nombre de la variable tipo puntero
  • Tipo_de_Dato es el tipo de base del puntero
  • El asterisco (*) usado para declarar el puntero, es el mismo usado para la multiplicación. Sin embargo, en esta declaración, el asterisco se esta utilizando para designar una variable como puntero
Ejemplo:
int *ip; // ip es un puntero a variables tipo int
float *fp; // fp es un puntero a variables tipo float
char *cp; // cp es un puntero a variables tipo char

Como ya se sabe, cada variable tiene una ubicación en la memoria y cada ubicación de la memoria tiene su dirección definida a la que se puede acceder utilizando el operador & que denota una dirección en la memoria

Hay algunas operaciones importantes que haremos con la ayuda de punteros con mucha frecuencia:

  • Definimos una variable puntero
  • Asignamos la dirección de una variable a un puntero
  • Finalmente accedemos al valor en la dirección disponible en la variable de puntero
Ejemplo: si declaramos las siguientes variables tenemos lo siguiente
int edad = 10; //variable tipo int
int* pr; // pr es un puntero a variables tipo int
Ejemplo:
int main(void) {
int edad; //variable tipo int
int *ptr; // ptr es un puntero a variables tipo int

ptr = &edad; // ptr apunta a la direccion de "edad"
}
Así:

Una vez que un puntero apunte a una variable cualquiera, se puede acceder al valor de dicha variable utilizando el nombre del puntero precedido por un asterisco, de esta forma:

int main(void) {
int edad; //variable tipo int
int *ptr; // ptr es un puntero a variables tipo int

ptr = &edad; // ptr apunta a la direccion de "edad"
*ptr = 10; // Esto es como la variable "edad" es igual a 10
}
De esta manera:

La expresión *prt se debería leer: “la variable apuntada por prt”.

En resumen:
  • Un puntero guarda direcciones de memoria. Así mismo, un puntero al ser una variable, tiene su propia dirección de memoria.
  • Con el puntero no solo accedemos al valor de la variable apuntada, sino que también podemos modificarlo
  • Un puntero obtiene su valor a partir de la referencia de una variable que se consigue con el operador &
Ejemplo:
int main(void) {
int height, width, n; //Variables tipo int
int * p, * q; // p y q son punteros a variables de tipo int

p = &height; // p apunta a height
*p = 10; // Esto es como height = 10
p = &width; // p apunta ahora a width
*p = 50; // Esto es como width = 50
height = *p; // Esto es como height = width
q = &height; // q apunta a height

n = (*p + *q)/2; // Esto es como n = (height + width)/2
}

Punteros con matrices

Un puntero a una matriz es simplemente un puntero al tipo de dato de la matriz. Cuando se asigna un puntero a una matriz, en realidad el puntero toma la dirección de su primer elemento, a menos que se especifique otro elemento. Luego, bastará con modificar el valor del puntero para que apunte a los otros elementos de la matriz. Veamóslo a través de un ejemplo:

int main(void) {
int m; //Variable tipo int
int * prt; // prt es un puntero a variables de tipo int
int Dig[] = {78, 98, 26}; // Matriz de variables tipo int

prt = &Dig; // prt apunta a Dig (a su primer elemento)
m = *prt; // Esto da m = 78
prt++; // Incrementa prt para apuntar a siguiente elemento
m = *prt; // Esto da m = 98
prt++; // Incrementa prt para apuntar a siguiente elemento
m = *prt; // Esto da m = 26

*prt = 10; // Con esto Dig[2] valdrá 10
prt--; // Decrementar prt para apuntar a elemento anterior
*prt = 100; // Con esto Dig[1] valdrá 100
prt = Dig; // prt apunta a Dig Es lo mismo que prt = &Dig
prt = NULL; // puntero nulo, Lo mismo que p = 0x0000, por lo tanto Dig[0] valdra 0
}
